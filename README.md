# Client Feedback Generator

A satirical Figma plugin that drops realistic (and brutally relatable) client feedback onto your designs as sticky notes — complete with severity levels, categories, and connector lines to the elements being "critiqued."

Ever wished a client would just say what they mean? Now you can simulate it yourself, for laughs, for stress-testing your designs, or for pure catharsis.

## What it does

Run the plugin on any selection (or the whole page), and it analyzes your design — checking for logos, buttons, text, colors, images, icons, gradients, frames, and components — then generates a batch of feedback notes pulled from a pool of classic client comments like:

- 🔴 **Panic** — "This font isn't working... wait, where's the call to action?"
- 🟡 **Change Request** — "Can we make the logo bigger?"
- ⚪ **Nitpick** — "There is WAY too much whitespace."

Each note is color-coded by severity, can be **dismissed** or marked **addressed**, and is placed directly on the canvas next to your design — with an optional connector line pointing to the relevant element (e.g. your logo, a button).

## Features

- **Design-aware feedback** — generators inspect your actual design (logo presence, button labels, text snippets, color palette, etc.) and tailor feedback accordingly
- **Severity levels** — Panic 🔴, Change Request 🟡, Nitpick ⚪
- **Status tracking** — mark notes as Active, Dismissed, or Addressed directly on the canvas
- **Connector lines** — visually link feedback to the specific node it's about
- **Persistent plugin data** — feedback state is stored on each sticky note frame, so it survives reloads
- **No network access** — fully local, no external API calls (`networkAccess: { allowedDomains: ["none"] }`)

## Installation (development)

1. Clone or download this repository
2. Open a terminal in the project folder and install dependencies:
   ```bash
   npm install
   ```
3. Build the plugin:
   ```bash
   npm run build
   ```
   This compiles `code.ts` → `dist/code.js` using the TypeScript compiler.
4. In Figma (desktop app): **Plugins → Development → Import plugin from manifest...**
5. Select the `manifest.json` file from this project folder.
6. Run it via **Plugins → Development → Client Feedback Generator**

## Development

```bash
npm run watch
```
Watches `code.ts` and recompiles automatically on save.

## How it works

1. `getDesignProfile()` scans the current selection (or page) and builds a profile of the design — logo presence, button labels, color usage, text content, etc.
2. `generateFeedback()` runs that profile through a pool of feedback generators, each producing a contextual (or generic fallback) comment.
3. `createAnnotations()` places each feedback item as a sticky note (`FrameNode`) on the canvas, color-coded by severity, with an optional connector line to the related node.
4. The UI (`ui.html`) lets you dismiss or address each item — these actions update the sticky note's appearance and persisted plugin data directly in Figma.
